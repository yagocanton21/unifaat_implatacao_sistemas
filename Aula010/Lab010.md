## Passo 1: Criar DB Subnet Group

### O que é DB Subnet Group?

Um **DB Subnet Group** define as subnets onde sua instância RDS será lançada. É necessário para:
- Determinar a topologia de rede do banco
- Configurar Multi-AZ (múltiplas zonas de disponibilidade)
- Garantir alta disponibilidade

### Criar Subnets para RDS

```bash
# Criar subnet 1 (Zona 1)
aws ec2 create-subnet \
  --vpc-id vpc-xxxxxxxx \
  --cidr-block 10.0.2.0/24 \
  --availability-zone us-east-1a \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=RDS-Subnet-1a}]'

# Criar subnet 2 (Zona 2)
aws ec2 create-subnet \
  --vpc-id vpc-xxxxxxxx \
  --cidr-block 10.0.3.0/24 \
  --availability-zone us-east-1b \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=RDS-Subnet-1b}]'

# Verificar subnets criadas
aws ec2 describe-subnets --filters "Name=tag:Name,Values=RDS-Subnet-*" --query 'Subnets[*].[SubnetId,AvailabilityZone,CidrBlock]' --output table
```

### Criar DB Subnet Group via AWS CLI

```bash
# Salvar IDs das subnets
SUBNET_1a="subnet-xxxxxxxx"  # RDS-Subnet-1a
SUBNET_1b="subnet-yyyyyyyy"  # RDS-Subnet-1b

# Criar DB Subnet Group
aws rds create-db-subnet-group \
  --db-subnet-group-name unifaat-db-subnet \
  --db-subnet-group-description "DB Subnet Group para RDS UniFAAT" \
  --subnet-ids $SUBNET_1a $SUBNET_1b \
  --tags Key=Name,Value=UniFAAT-DBSubnetGroup

# Verificar criação
aws rds describe-db-subnet-groups --db-subnet-group-name unifaat-db-subnet
```

## Passo 2: Configurar Security Group para RDS

### O que é um Security Group para RDS?

Um **Security Group** para RDS funciona como firewall, permitindo apenas conexões autorizadas ao banco de dados. Deve permitir:
- Conexões apenas da aplicação (EC2)
- Porta específica do banco de dados
- Negar tudo que não for explicitamente autorizado

### Criar Security Group para RDS

```bash
# Criar security group para RDS
aws ec2 create-security-group \
  --group-name RDS-SG \
  --description "Security Group para RDS - UniFAAT" \
  --vpc-id vpc-xxxxxxxx \
  --tag-specifications 'ResourceType=security-group,Tags=[{Key=Name,Value=RDS-SG}]'

# Salvar o grupo ID
RDS_SG="sg-xxxxxxxx"

# Permitir conexões MySQL (porta 3306) do EC2 Web Server
aws ec2 authorize-security-group-ingress \
  --group-id $RDS_SG \
  --protocol tcp \
  --port 3306 \
  --source-security-group-id sg-webserver \
  --description "MySQL from WebServer-SG"

# Verificar regras
aws ec2 describe-security-groups --group-ids $RDS_SG --query 'SecurityGroups[0].IpPermissions'
```

### Criar Security Group via Console

**1. EC2 Dashboard:**
- Acesse EC2 → Security groups

**2. Create Security Group:**

**3. Configurar:**
- Security group name: `RDS-SG`
- Description: `Security Group para RDS - UniFAAT`
- VPC: Selecione sua VPC

**4. Inbound Rules:**
- Type: MySQL/Aurora
- Port: 3306
- Source: Selecione `Web-Server-SG`

**5. Create:**

## Passo 3: Criar Instância RDS

### Opções de Engine RDS

| Engine | Melhor para | Custo | Compatibilidade |
|--------|-----------|-------|--|
| **MySQL 8.0** | Apps web, padrão | Menor | Alta (MariaDB, PostgreSQL similar) |
| **PostgreSQL** | Apps complexas, dados avançados | Médio | Excelente (JSON, arrays) |
| **MariaDB** | Alternativa MySQL | Menor | Muito compatível com MySQL |
| **Aurora MySQL** | Alta escala, ótima performance | Maior | MySQL compatível |

### Criar RDS via AWS CLI

```bash
# Definir variáveis
DB_INSTANCE_ID="unifaat-mysql-db"
DB_SUBNET_GROUP="unifaat-db-subnet"
RDS_SG="sg-xxxxxxxx"
MASTER_USER="admin"
MASTER_PASSWORD="UniFAAT@2026Admin!"  # ALTERAR EM PRODUÇÃO!

# Criar instância RDS
aws rds create-db-instance \
  --db-instance-identifier $DB_INSTANCE_ID \
  --db-instance-class db.t3.micro \
  --engine mysql \
  --engine-version "8.0.35" \
  --master-username $MASTER_USER \
  --master-user-password $MASTER_PASSWORD \
  --allocated-storage 20 \
  --storage-type gp2 \
  --db-subnet-group-name $DB_SUBNET_GROUP \
  --vpc-security-group-ids $RDS_SG \
  --multi-az \
  --backup-retention-period 7 \
  --preferred-backup-window "03:00-04:00" \
  --preferred-maintenance-window "sun:04:00-sun:05:00" \
  --enable-cloudwatch-logs-exports '["error","general","slowquery"]' \
  --tags Key=Name,Value=UniFAAT-MySQL-DB Key=Course,Value=ADS

# Monitorar status
aws rds describe-db-instances \
  --db-instance-identifier $DB_INSTANCE_ID \
  --query 'DBInstances[0].[DBInstanceIdentifier,DBInstanceStatus,DBInstanceClass,Engine]' \
  --output table

# Aguardar até status = "available"
# Pode levar 5-15 minutos
```

### Criar Tabelas Mais Complexas

```sql
-- Tabela de Disciplinas
CREATE TABLE disciplinas (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  professor VARCHAR(100),
  creditos INT,
  semestre INT,
  UNIQUE(nome)
);

-- Tabela de Matrículas (relacionamento muitos-para-muitos)
CREATE TABLE matriculas (
  id INT AUTO_INCREMENT PRIMARY KEY,
  aluno_id INT NOT NULL,
  disciplina_id INT NOT NULL,
  data_matricula DATE DEFAULT CURDATE(),
  nota DECIMAL(4,2),
  FOREIGN KEY (aluno_id) REFERENCES alunos(id) ON DELETE CASCADE,
  FOREIGN KEY (disciplina_id) REFERENCES disciplinas(id) ON DELETE CASCADE,
  UNIQUE(aluno_id, disciplina_id)
);

-- Inserir disciplinas
INSERT INTO disciplinas (nome, professor, creditos, semestre) VALUES
('Implementação de Software', 'Prof. João', 4, 2),
('Banco de Dados', 'Prof. Maria', 4, 2),
('Algoritmos', 'Prof. Pedro', 3, 1);

-- Matricular aluno
INSERT INTO matriculas (aluno_id, disciplina_id) VALUES
(1, 1),  -- João em Implementação
(2, 1),  -- Maria em Implementação
(1, 2);  -- João em Banco de Dados

-- Query com JOIN
SELECT 
  a.nome as aluno,
  d.nome as disciplina,
  m.data_matricula,
  m.nota
FROM matriculas m
JOIN alunos a ON m.aluno_id = a.id
JOIN disciplinas d ON m.disciplina_id = d.id
ORDER BY a.nome, d.nome;
```

## Passo 7: Troubleshooting de RDS

### Problema: "Connection refused"

**Sintomas:**
```
ERROR 2003 (HY000): Can't connect to MySQL server on 'unifaat-mysql-db.xxx.rds.amazonaws.com' (111)
```

**Causas possíveis:**
1. Instância RDS não está em status "available"
2. Security Group não permite porta 3306
3. RDS em subnet diferente de EC2

**Diagnóstico:**
```bash
# 1. Verificar status do RDS
aws rds describe-db-instances \
  --db-instance-identifier unifaat-mysql-db \
  --query 'DBInstances[0].DBInstanceStatus'

# 2. Verificar regras de Security Group
aws ec2 describe-security-groups \
  --group-ids sg-xxxxxxxx \
  --query 'SecurityGroups[0].IpPermissions'

# 3. Testar conectividade (do EC2)
nc -zv unifaat-mysql-db.cxxxxxxxxxxxx.us-east-1.rds.amazonaws.com 3306
```

**Solução:**
```bash
# Adicionar regra ao Security Group
aws ec2 authorize-security-group-ingress \
  --group-id sg-rds \
  --protocol tcp \
  --port 3306 \
  --source-security-group-id sg-webserver
```

### Problema: "Access denied for user 'admin'"

**Sintomas:**
```
ERROR 1045 (28000): Access denied for user 'admin'@'10.0.1.x' (using password: YES)
```

**Causas:**
1. Senha incorreta
2. Usuário não existe
3. Permissões insuficientes

**Diagnóstico:**
```bash
# Listar usuários no RDS
mysql -h <RDS_ENDPOINT> -u admin -p -e "SELECT User, Host FROM mysql.user;"

# Verificar permissões
mysql -h <RDS_ENDPOINT> -u admin -p -e "SHOW GRANTS FOR 'admin'@'%';"
```

**Solução:**
```bash
# Resetar senha do usuário master
aws rds modify-db-instance \
  --db-instance-identifier unifaat-mysql-db \
  --master-user-password "NewPassword123!" \
  --apply-immediately
```

### Problema: "Out of memory" ou lentidão

**Sintomas:**
- Queries lentas
- Erros de timeout
- Alta latência

**Diagnóstico:**
```bash
# Ver informações de performance
mysql> SHOW PROCESSLIST;
mysql> SHOW GLOBAL STATUS LIKE 'Threads%';

# Ver tamanho do banco
SELECT table_schema, 
       ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) as MB
FROM information_schema.tables
GROUP BY table_schema;
```

**Solução:**
```bash
# Fazer upgrade de instância
aws rds modify-db-instance \
  --db-instance-identifier unifaat-mysql-db \
  --db-instance-class db.t3.small \
  --apply-immediately

# Otimizar tabelas
mysql> OPTIMIZE TABLE alunos, disciplinas, matriculas;
```

### Problema: Replicação Multi-AZ falhando

**Sintomas:**
- Failover não funciona
- Réplica fora de sincronismo

**Diagnóstico:**
```bash
# Verificar status Multi-AZ
aws rds describe-db-instances \
  --db-instance-identifier unifaat-mysql-db \
  --query 'DBInstances[0].[MultiAZ,PendingModifiedValues.MultiAZ]'

# Verificar logs de erro
aws rds describe-db-log-files \
  --db-instance-identifier unifaat-mysql-db
```

**Solução:**
```bash
# Habilitar Multi-AZ
aws rds modify-db-instance \
  --db-instance-identifier unifaat-mysql-db \
  --multi-az \
  --apply-immediately
```

### Logs para Troubleshooting

## 🚀 Conceitos Avançados

### RDS vs EC2 com MySQL Instalado

| Aspecto | RDS Gerenciado | EC2 + MySQL Manual |
|---------|--------|-----------------|
| **Backup** | Automático (7 dias) | Manual, você controla |
| **Updates** | AWS gerencia | Você instala/testa |
| **High Availability** | Multi-AZ integrado | Você configura |
| **Monitoramento** | CloudWatch integrado | Você instala/configura |
| **Performance** | Otimizado por padrão | Depende da config |
| **Segurança** | Patches automáticos | Manual |
| **Custo** | Mais caro | Mais barato |
| **Complexidade** | Simples | Complexa |

### Multi-AZ vs Read Replicas

**Multi-AZ (Alta Disponibilidade):**
- 1 primary + 1 standby (diferentes AZs)
- Sincronização síncrona (garantida)
- Failover automático (<2 min)
- Mesmo banco de dados
- Melhor para: **Produção**

**Read Replicas (Escala de Leitura):**
- 1 primary + N replicas
- Sincronização assíncrona (eventual)
- Failover manual
- Bancos diferentes (replica atrasada)
- Melhor para: **Relatórios, Analytics**

### Otimização de Queries

## 🧹 Procedimento de Limpeza

### ⚠️ IMPORTANTE: Evitar Custos

RDS gera custos mesmo parado. Deletar recursos que não usa mais!

### Limpeza Completa (na ordem)

**1. Deletar Snapshots:**
```bash
# Listar snapshots
aws rds describe-db-snapshots \
  --db-instance-identifier unifaat-mysql-db \
  --query 'DBSnapshots[*].[DBSnapshotIdentifier,SnapshotCreateTime]'

# Deletar snapshot manual
aws rds delete-db-snapshot \
  --db-snapshot-identifier unifaat-mysql-db-manual-backup
```

**2. Deletar Instância RDS:**
```bash
# Deletar SEM final snapshot (cuidado!)
aws rds delete-db-instance \
  --db-instance-identifier unifaat-mysql-db \
  --skip-final-snapshot

# Ou COM final snapshot (seguro)
aws rds delete-db-instance \
  --db-instance-identifier unifaat-mysql-db \
  --final-db-snapshot-identifier unifaat-final-backup
```

**3. Deletar DB Subnet Group:**
```bash
# Listar
aws rds describe-db-subnet-groups

# Deletar (somente se não em uso)
aws rds delete-db-subnet-group \
  --db-subnet-group-name unifaat-db-subnet
```

**4. Deletar Security Group:**
```bash
# Listar
aws ec2 describe-security-groups \
  --filters "Name=group-name,Values=RDS-SG"

# Deletar
aws ec2 delete-security-group --group-id sg-xxxxxxxx
```

**5. Deletar Subnets (opcional):**
```bash
# Listar
aws ec2 describe-subnets \
  --filters "Name=tag:Name,Values=RDS-Subnet-*"

# Deletar
aws ec2 delete-subnet --subnet-id subnet-xxxxxxxx
```
